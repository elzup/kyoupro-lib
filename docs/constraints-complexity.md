# 制約 → 計算量 → アルゴリズム対応表

問題を読んだら最初に N の制約を見て、許される計算量からアルゴリズムを絞る。
目安: **10⁸ 回/秒** (Kotlin は定数倍が重いので 10⁷〜10⁸ を意識)。

| N の上限 | 許される計算量 | 候補アルゴリズム |
| --- | --- | --- |
| N ≤ 10 | O(N!) | 順列全列挙 |
| N ≤ 16 | O(2ᴺ·N) 〜 O(2ᴺ·N²) | bit全探索、bitDP (TSP) |
| N ≤ 40 | O(2^(N/2)) | 半分全列挙 |
| N ≤ 100 | O(N⁴) | DP (3重ループ + 遷移) |
| N ≤ 500 | O(N³) | Warshall-Floyd、区間DP |
| N ≤ 3,000 | O(N²) | 素朴な DP、全ペア列挙 |
| N ≤ 10⁵ | O(N√N) | クエリ平方分割 |
| N ≤ 2×10⁵ | O(N log N) | ソート、セグ木/BIT、Dijkstra、LIS |
| N ≤ 10⁷ | O(N) | 累積和、しゃくとり法、篩 |
| N ≤ 10⁹ | O(√N) | 素数判定、約数列挙 |
| N ≤ 10¹⁸ | O(log N) | 二分探索、modpow、桁DP、行列累乗 |

## 値の上限とオーバーフロー

| 型 | 上限 | 目安 |
| --- | --- | --- |
| Int | 2.1×10⁹ | 10⁵ × 10⁵ の積で**もう溢れる** |
| Long | 9.2×10¹⁸ | 10⁹ × 10⁹ までOK。和なら 10¹⁸ |
| Double | 仮数 53bit | 10¹⁵ を超える整数は誤差が出る |

- 迷ったら Long。`1 shl 40` は Int 演算で壊れる → `1L shl 40`
- 幾何や確率以外で Double を使わない (二分探索は整数で回す)

## クエリ問題の定石

| クエリ内容 | 構造 |
| --- | --- |
| 点更新 + 区間和 | BIT |
| 点更新 + 区間 min/max | セグ木 |
| 区間加算 + 最後に全点出力 | いもす法 (差分配列) |
| 連結判定 (辺の追加のみ) | Union-Find |
| 最小値の取り出し | PriorityQueue |
| 値の挿入 + 順序付き検索 | TreeMap / TreeSet |
