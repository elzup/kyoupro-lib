## 0. 問題（要約）

- N 人の容疑者。人 i は時刻 `[S_i, T_i]` に在館。
- 犯行は長さ D。整数時刻 x に開始し x+D に完了。**2 人の共犯者は犯行の全区間 `[x, x+D]` に在館**している必要がある。
- 「**(共犯者ペア, 開始時刻 x) の組合せ数**」を求める。ペアの順序は区別しない。
- 制約: `2 ≤ N ≤ 2×10⁵`, `1 ≤ S_i ≤ T_i ≤ 10⁶`, `1 ≤ D ≤ 10⁶`。

サンプル: `4 / 0 / 5999994`。

## 2. キモ（転換点）

この問題の本質は **「ペアを数える」を「時刻ごとに在館人数を数える」に裏返す**こと。

1. **人 i を“開始時刻の区間”に変換**（#10）
   人 i が全区間 `[x, x+D]` に在館 ⇔ `S_i ≤ x` かつ `x+D ≤ T_i` ⇔ **`x ∈ [S_i, T_i − D]`**。
   → 各人を「犯行開始時刻として使える区間」に読み替える。`T_i − D < S_i` なら空（その人は単独でも犯人になれない）。

2. **各 x で C(c(x), 2) を足す**
   開始時刻 x を取れる人数を `c(x)` とすると、その x でのペア数は `c(x)·(c(x)−1)/2`。
   答え = **`Σ_x C(c(x), 2)`**。ペアごとに重複は出ない（各 (ペア, x) を x 側から 1 回ずつ数える）。

3. **`c(x)` は いもす法（差分配列）**（#9）
   区間 `[S_i, T_i−D]` を `diff[S_i]++`, `diff[T_i−D+1]--` で重ね、前計算 → x を 0..10⁶ 走査して `C(c,2)` を加算。
   計算量 **O(N + maxT)**。N² の 4×10¹⁰（#5 で却下）を回避できる。

> #6 の「D 窓の min を合計」は惜しい寄り道。正しくは **min ではなく `[S_i, T_i−D]` への変換 ＋ nC2**。
> 「滞在人数を時刻軸で見る」という #6 の発想自体（#3 のガントチャート）は本筋で、定式化だけ #10 で直った。

## 4. つまずき・寄り道の分析

| 寄り道 | なぜ起きたか | 正したきっかけ |
| ------ | ------------ | -------------- |
| 区間 DP?（#4） | 「大きな数＝DP」の短絡。実際は数え上げを分解すれば閉じた式。 | 計算量と構造を見て DP 不要と判断。 |
| D 窓の min を合計（#6） | 「滞在人数」までは正しいが、ペア数の式化を min と取り違え。 | `[S_i,T_i−D]` 変換に気づき nC2 へ（#10）。 |

## 5. 完成コード（Kotlin）

```kotlin
fun main() {
    val out = PrintWriter(System.out.bufferedWriter())

    val (n, d) = nextInts(2)

    val sums = IntArray(1000000 + 10) { 0 }
    for (i in 0 until n) {
        val s = nextInt()
        val t = nextInt()
        if (t - s < d) continue   // [s, t-d] が空（単独でも犯人になれない）→ 無視
        sums[s]++
        sums[t - d + 1]--         // d=2 のとき開始可能は [s, t-1] なので t-d+1 で閉じる
    }

    var cur = 0
    var ans = 0L
    for (x in 0..1000000) {
        cur += sums[x]
        ans += cur.toLong() * (cur - 1) / 2 // nC2
    }

    out.println(ans)
    out.flush()
}
```

実物: `atcoder/abc462/d/Main.kt`。サンプル 3 つ AC（`./run.sh abc462/d`）。

### サンプル 1 のトレース（N=3, D=2）

| 人 | `[S, T]` | 開始可能 `[S, T−D]` |
| -- | -------- | ------------------- |
| 1 | [9, 17] | [9, 15] |
| 2 | [10, 12] | [10, 10] |
| 3 | [13, 20] | [13, 18] |

x=10 で c=2（人1,2）→ 1、x=13,14,15 で c=2（人1,3）→ 各 1。計 **4**。✓

## 6. tips チェックリスト（このセッションで効いたもの）

- [ ] **変数名を揃える**（`s`/`t`/`d`、`cur`/`ans`）— 添字や役割の取り違えを防ぐ。
- [ ] **添字で混乱しない** — いもす の `-1` を置く位置（`t-d+1`）は具体値（d=2）で 1 回手で確かめる。
- [ ] **0 を無視しても成り立つ** — `S_i ≥ 1` なので index 0 は未使用。`C(0,2)=C(1,2)=0` だから人数 0/1 の x も特別扱い不要。
- [ ] **オフラインライブラリ**（Mac: Dash / Zeal）— 標準ライブラリ・文法の即引き用に準備。
- [ ] **オーバーフロー** — ペア数は最大 `C(2×10⁵,2)×10⁶ ≒ 2×10¹⁶`。`ans` は必ず Long、`cur.toLong()*(cur-1)` で乗算前に昇格。
